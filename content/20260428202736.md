## 为什么你需要掌握 AI 术语

人工智能领域发展迅猛，新概念、新术语层出不穷。无论你是开发者、产品经理，还是对 AI 感兴趣的入门者，理解核心术语都是跟上行业节奏的第一步。本文系统梳理了 AI 领域最关键的专业术语，按类别分层讲解，帮助你建立完整的知识框架。

## 基础概念类

这些术语是理解 AI 的基石，几乎是所有进阶讨论的起点。

- **人工智能（Artificial Intelligence, AI）**：让机器模拟人类智能行为的技术总称，包括感知、推理、学习和决策等能力
- **机器学习（Machine Learning, ML）**：AI 的核心子领域，通过数据训练模型，使机器自动发现规律并做出预测，而非依赖人工编写规则
- **深度学习（Deep Learning, DL）**：机器学习的分支，使用多层神经网络自动提取数据特征，在图像、语音、自然语言等领域表现突出
- **算法（Algorithm）**：解决特定问题的计算步骤和规则集合，是 AI 模型运行的底层逻辑
- **模型（Model）**：通过算法在数据上训练得到的数学表示，用于对新数据进行预测或生成
- **训练（Training）**：用大量数据反复调整模型参数的过程，目标是让模型的预测结果尽可能接近真实值
- **推理（Inference）**：训练完成后，用模型对新输入数据进行预测或生成的过程

**过拟合（Overfitting）** 是训练中常见的陷阱——模型在训练数据上表现极好，却无法泛化到新数据。解决方法包括增加数据量、使用正则化、提前停止训练等。

## 神经网络与架构类

神经网络是当代 AI 的核心引擎，不同架构决定了模型擅长的任务类型。

- **神经网络（Neural Network）**：模拟生物神经元连接方式的计算模型，由输入层、隐藏层和输出层组成
- **卷积神经网络（CNN）**：擅长处理图像数据，通过卷积核提取空间特征，广泛用于图像分类、目标检测
- **循环神经网络（RNN）**：擅长处理序列数据（如文本、时间序列），具有记忆先前输入的能力
- **Transformer**：当前最主流的架构，基于自注意力机制（Self-Attention），能并行处理序列中所有位置的关系，是 GPT、BERT 等大模型的基础
- **注意力机制（Attention Mechanism）**：让模型在处理输入时，动态聚焦于最相关的部分，显著提升了序列建模的效果
- **生成对抗网络（GAN）**：由生成器和判别器对抗训练，擅长生成逼真的图像、音频等数据

> **注意**：Transformer 架构自 2017 年提出以来，已经几乎统一了自然语言处理领域，并在计算机视觉（ViT）、音频处理等领域展现出强大潜力，是当前 AI 最重要的基础架构。

## 大模型与 NLP 类

随着 ChatGPT 的爆火，大模型相关术语成为最受关注的知识点。

- **大语言模型（Large Language Model, LLM）**：参数量达数十亿甚至万亿的语言模型，具备强大的文本理解与生成能力，代表产品包括 GPT-4、Claude、GLM 等
- **提示词（Prompt）**：输入给大模型的指令或问题，Prompt 的质量直接决定输出效果
- **提示词工程（Prompt Engineering）**：设计和优化提示词的技术，通过结构化、示例引导等方式提升模型输出质量
- **上下文学习（In-Context Learning）**：在提示词中提供少量示例，让模型在不修改参数的情况下学会新任务
- **微调（Fine-tuning）**：在预训练模型基础上，用特定领域数据继续训练，使模型适配特定任务
- **RAG（检索增强生成）**：先从外部知识库检索相关信息，再让模型基于检索结果生成回答，有效减少幻觉问题
- **Token**：模型处理文本的最小单位，一个 Token 大致对应 0.5～1 个中文字或 1 个英文单词
- **幻觉（Hallucination）**：模型生成看似合理但事实上错误的内容，是大模型落地应用的主要风险之一
- **思维链（Chain of Thought, CoT）**：通过引导模型逐步推理，将复杂问题分解为多个中间步骤，显著提升逻辑推理能力
- **Agent（智能体）**：能够自主感知环境、制定计划、调用工具并执行动作的 AI 系统是大模型从"对话工具"走向"行动者"的关键形态

## 训练与优化类

理解这些术语，能帮助你把握模型性能调优的核心逻辑。

- **参数（Parameters）**：模型内部可学习的数值，训练过程就是不断调整参数的过程
- **超参数（Hyperparameters）**：训练前手动设定的配置项，如学习率、批大小、训练轮数等
- **学习率（Learning Rate）**：控制每次参数更新步幅的大小，学习率过大导致训练不稳定，过小则收敛缓慢
- **损失函数（Loss Function）**：衡量模型预测值与真实值差距的函数，训练目标就是最小化损失
- **梯度下降（Gradient Descent）**：根据损失函数的梯度方向更新参数的优化算法，是深度学习最常用的优化方法
- **批量大小（Batch Size）**：每次训练迭代使用的数据样本数量
- **Epoch**：完整遍历一次全部训练数据的过程
- **正则化（Regularization）**：通过添加约束项防止模型过拟合的技术，常见的有 L1、L2 正则化
- **分布式训练（Distributed Training）**：将训练任务分散到多个 GPU 或机器上并行执行，是大模型训练的必选项

## 评估与指标类

- **准确率（Accuracy）**：正确预测数占总样本数的比例
- **精确率（Precision）**：预测为正的样本中，实际为正的比例
- **召回率（Recall）**：实际为正的样本中，被正确预测为正的比例
- **F1 Score**：精确率和召回率的调和平均数，综合评估模型表现
- **BLEU**：评估机器翻译或文本生成质量的指标，通过对比生成文本与参考文本的 n-gram 重合度计算
- **Perplexity（困惑度）**：衡量语言模型对文本预测能力的指标，值越低表示模型越好
- **基准测试（Benchmark）**：在标准数据集上评估模型性能的测试集，如 MMLU、HumanEval 等

## 代码演示

以下代码展示了如何使用 Python 计算分类任务中精确率、召回率和 F1 Score，帮助你直观理解这些评估指标。

```js
function computeMetrics(tp, fp, fn) {
  const precision = tp / (tp + fp);
  const recall = tp / (tp + fn);
  const f1 = (2 * precision * recall) / (precision + recall);

  return {
    precision: precision.toFixed(4),
    recall: recall.toFixed(4),
    f1Score: f1.toFixed(4),
  };
}

// 示例：某疾病检测模型，100个真阳性，20个假阳性，30个假阴性
const result = computeMetrics(100, 20, 30);
console.log("精确率 (Precision):", result.precision);
console.log("召回率 (Recall):", result.recall);
console.log("F1 Score:", result.f1Score);

// 输出：
// 精确率 (Precision): 0.8333
// 召回率 (Recall): 0.7692
// F1 Score: 0.8000
```
